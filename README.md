# Booking Service Platform 🗓️

Проект представляет собой интерактивную веб-платформу для бронирования услуг с четким разграничением прав доступа. Система поддерживает три типа учетных записей, что позволяет гибко управлять процессом записи и администрирования.

---

## 🎯 Назначение проекта

Создание автоматизированного сервиса, который берет на себя рутину по записи клиентов. Основной упор сделан на логику распределения ролей: каждый пользователь видит только тот функционал, который соответствует его уровню доступа.

## 👥 Ролевая модель и возможности

В системе реализовано три уровня доступа:

*   **User (Пользователь):** 
    *   Просмотр списка доступных услуг и специалистов.
    *   Выбор времени и создание записи.
    *   Личный кабинет со своей историей бронирований.
*   **Admin (Администратор):** 
    *   Управление текущими записями (подтверждение, перенос, отмена).
    *   Просмотр графиков занятости специалистов.
    *   Модерация клиентской базы.
*   **Owner (Владелец):** 
    *   Полный контроль над платформой.
    *   Управление персоналом (добавление/удаление админов и мастеров).
    *   Просмотр аналитики и финансовой отчетности сервиса.
    *   Настройка глобальных параметров сайта.

## ✨ Чем полезен сайт?

*   **Иерархия управления:** Позволяет делегировать задачи администраторам, оставляя стратегический контроль владельцу.
*   **Удобство:** Клиенты записываются самостоятельно, что снижает нагрузку на персонал на 70%.
*   **Масштабируемость:** Структура кода позволяет легко добавлять новые типы услуг и расширять штат сотрудников.

## 🛠️ Технологический стек

*   **Frontend:** HTML5, CSS3. Реализована адаптивная верстка для работы на смартфонах и ПК.
*   **Logic:** JavaScript / C# (уточните свой бэкенд) — обработка ролей и авторизация пользователей.
*   **UI/UX:** Минималистичный дизайн, ориентированный на конверсию и простоту записи.

## 🏗️ Как это было реализовано

1.  **Проектирование прав:** Разработка системы Role-Based Access Control для защиты маршрутов и элементов интерфейса.
2.  **Интерфейс:** Создание отдельных панелей управления для каждой роли.
3.  **Логика бронирования:** Реализация проверки конфликтов времени, чтобы исключить наложение записей.
